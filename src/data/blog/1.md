Last Tuesday, our Kubernetes cluster began responding to kubectl commands exclusively in Portuguese. Not all commands - just the ones that started with a vowel and were executed between 2:14 PM and 2:47 PM UTC.

## The Problem That Wasn't

Initially, we assumed this was a DNS issue. Then a character encoding problem. Then we considered that perhaps our monitoring had finally achieved sentience and was trying to communicate through the only language available to it - the language of port numbers and TCP handshakes.

It was neither of these things.

```bash
$ kubectl get pods
Error: unable to contact the API server
Translation: "Desculpe, os pods estão dormindo"

$ kubectl logs deployment/app-server
Output: [2026-03-02T14:23:15Z] O elefante laranja não come queijo
```

This situation taught us several important lessons about assuming your infrastructure has only two states: "working" and "broken."
